# 스크럼 가이드

## 개요
LXP 프로젝트의 스크럼 프로세스 가이드입니다. 팀의 효율적인 협업과 지속적인 개선을 위한 규칙과 절차를 정의합니다.

## 스크럼 이벤트

### 1. 데일리 스탠드업 (Daily Standup)

**시간**: 매일 오전 10:00 (15분)
**참석자**: 팀 전체
**장소**: 온라인/오프라인 (팀 상황에 따라)

#### 진행 방식

각 팀원이 순서대로 다음 3가지 질문에 답변:

1. **어제 무엇을 했나?**
   - 완료한 작업
   - 예: "enrollment API 테스트 코드 작성 완료"

2. **오늘 무엇을 할 것인가?**
   - 계획된 작업
   - 예: "결제 모듈 통합 테스트 진행"

3. **장애물이 있는가?**
   - 블로커, 도움 필요한 사항
   - 예: "외부 결제 API 문서가 불명확해서 확인 필요"

#### 규칙
- ⏰ **15분 엄수**: 길어지면 별도 미팅
- 🎯 **문제 해결 X**: 문제 공유만, 해결은 별도 진행
- 💬 **간결하게**: 핵심만 전달
- 🚫 **보고 아님**: 관리자를 위한 보고가 아닌 팀 동기화
